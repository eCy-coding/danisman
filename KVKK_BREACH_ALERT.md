**Date:** 2026-05-29

## Breach Description

`server/routes/newsletter.ts` stores the raw client IP address in the
`NewsletterSubscriber` database table without hashing or minimization.

KVKK Art. 3(1)(d) classifies IP addresses as personal data ("kişisel veri").
The same file also stores `userAgent` (which may contribute to fingerprinting).
The contact route (`server/routes/contact.ts:118-120`) correctly SHA-256 hashes
the IP before storing, but the newsletter route does not.

## Fix Applied

Added `import crypto from 'node:crypto'` to `server/routes/newsletter.ts` and
replaced raw IP capture with SHA-256 hashed + truncated value, matching the
pattern in `contact.ts`.

```diff
+import crypto from 'node:crypto';
...
-      const ip = req.ip ?? null;
+      const ip = req.ip
+        ? crypto.createHash('sha256').update(req.ip).digest('hex').slice(0, 16)
+        : null;
```
