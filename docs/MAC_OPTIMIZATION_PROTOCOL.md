
# 🍏 MacBook Developer Optimization Protocol (2025)

**Objective:** Maintain maximum performance and available storage on a developer MacBook by eliminating "System Data" waste and package manager bloat.

## 1. Storage Diagnostics (Detecting the Waste)
Before cleaning, identify *what* is taking space.
- **Terminal:** Install `ncdu` or `dust` for a visual breakdown.
  ```bash
  du -sh ~/.npm ~/.pnpm-store ~/Library/Developer/Xcode/DerivedData ~/Library/Containers/com.docker.docker
  ```
- **Visual:** Use **GrandPerspective** (Free) or **DaisyDisk** for a heat map of your drive.

## 2. Package Managers (The Usual Suspects)
Package managers cache *every* version of *every* package you've ever downloaded.
### NPM
- **Command:** `npm cache clean --force`
- **Impact:** Clears the `~/.npm` folder (often 5-10GB+).

### Docker
- **Command:** `docker system prune -af --volumes`
- **Warning:** This deletes ALL stopped containers, unused images, and **volumes**.
- **Impact:** Can recover 20-50GB on active dev machines.

### PNPM (Recommended)
- **Status:** You are already using this locally.
- **Command:** `pnpm store prune`
- **Impact:** Removes unreferenced packages from the global content-addressable store.

## 3. Xcode & iOS Development
Even if you don't write Swift, installing Node/Python tools often creates Xcode artifacts.
- **Derived Data:** `rm -rf ~/Library/Developer/Xcode/DerivedData/*`
- **Simulators:** `xcrun simctl delete unavailable` (Removes runtimes for old iOS versions).

## 4. System Data & Git
- **Git Garbage Collection:** `git gc --prune=now --aggressive` (Run inside large repos).
- **Time Machine Snapshots:** Local snapshots can take space.
  - List: `tmutil listlocalsnapshots /`
  - Delete: `tmutil deletelocalsnapshots <date>`

## 5. Strategy for "Zero Waste" Projects
To ensure *future* projects stay small:
1.  **Use PNPM Enforced:** Set `engine-strict=true` in `.npmrc`.
2.  **Aggressive .gitignore:** Ensure `dist`, `coverage`, `node_modules` are ignored.
3.  **Cloud Build:** Push builds to Vercel/Netlify instead of building locally for production artifacts.
