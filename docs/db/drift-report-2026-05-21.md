## Tooling defects found & fixed

1. **`scripts/check-migration-drift.mjs` used Prisma-6 flags removed in Prisma 7.**
   `--to-schema-datamodel` / `--from-schema-datamodel` â†’ renamed to `--to-schema` / `--from-schema`. The dead flags made the CLI exit `1` (error), which the old classifier reported as **false-positive drift**. Fixed.

2. **`--exit-code` semantics changed in Prisma 7** (empty=0, error=1, **not-empty=2**). The old wrapper treated any non-zero as drift. Now: `0`â†’in-sync, `2`â†’drift, `1`â†’`cli_error` (skipped, not asserted as drift).
